DESCRIÇÃO DO PROJETO PARA TURMA

IESPFLIX | Portal do Projeto Web

Conexão do Frontend React/HTML com o seu Backend Spring Boot

Visão Geral

O projeto consiste no desenvolvimento completo de um ecossistema de gerenciamento cinematográfico (IESPFLIX) utilizando arquitetura cliente-servidor.

Exigências Técnicas do Professor:

  • Persistência de Dados: Configuração do Spring Data JPA com banco mapeado adequadamente.
  • Camada DTO (Data Transfer Object): Proteção e isolamento das entidades de banco de dados durante a transferência.
  • Consumo de APIs Externas: Integração dinâmica com a BrasilAPI para captura automática de CEP.
  • Clean Code: Utilização do Lombok para reduzir código boilerplate (Getters, Setters, Mappers organizados).

Sugestões & Boas Práticas

Dicas explícitas fornecidas para atingir os critérios máximos de avaliação:

  • Tratamento de Status HTTP: Toda operação de inserção bem-sucedida deve retornar o código de status HTTP 201 Created.
  • Regras de Usuário: O campo de e-mail deve possuir validação robusta (não nulo, único e formato de e-mail padrão).
  • Separação de Modelos: Evitar tabelas genéricas duplicadas. Cada recurso (/usuarios, /ingressos, /sessoes) deve manipular seu respectivo repositório físico.
  • Tratamento de Exceções: Retornos amigáveis em formato JSON para requisições inválidas feitas pelo cliente.

Endpoints da API

Selecione um dos caminhos obrigatórios exigidos no escopo do projeto para interagir com o seu IntelliJ local:

POST/usuarios

Cadastra um novo usuário e preenche o endereço automaticamente pelo CEP via BrasilAPI.

GET/filmes/ordenado

Retorna a lista completa de filmes cadastrados em ordem alfabética no seu banco de dados.

POST/ingressos

Efetua a reserva e persistência de bilhetes vinculados à tabela independente de ingressos.
